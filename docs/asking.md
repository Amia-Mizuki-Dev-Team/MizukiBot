# 提问的智慧

> “Bot 炸了”不是一个有效的问题。有效反馈应该让维护者能复现、定位和判断优先级。

<br>

<div class="mzk-card mzk-card--red">
  <h3>糟糕的提问</h3>
  
<div class="mzk-table-wrapper">

| 示例 | 为什么糟糕？ |
| :--- | :--- |
| “Bot 死了” | 没有描述具体情况：无响应、报错、超时、乱码还是权限问题。 |
| “指令没用” | 没有说明哪个指令、在哪个群、什么时候触发、Bot 返回了什么。 |
| “你自己看” | 没有给出可定位的信息，维护者需要重新追问。 |
| 只有一张模糊拍屏 | 难以看清报错信息，也不方便复制关键文本。 |

</div>
</div>

<div class="mzk-card mzk-card--green">
  <h3>有效提问三步</h3>

  <h4>1. 先确认基础条件</h4>
  <p>检查指令拼写、空格、区服前缀、Bot 权限、群聊是否禁言、是否开启了需要的群消息权限。</p>

  <h4>2. 提供完整上下文</h4>
  
<div class="mzk-table-wrapper">

| 信息 | 说明 |
| :--- | :--- |
| **发生了什么** | “我发送了 `完整指令`，预期 `结果`，实际 `情况`。” |
| **触发环境** | 群聊或私聊、Bot 账号、服务器区服、是否使用官方 Bot。 |
| **时间点** | 写清楚大概时间，方便查日志。 |
| **截图或文本** | 优先使用系统截图；报错文本请尽量复制原文。 |
| **复现步骤** | 从第 1 步到出错的完整流程。 |

</div>

  <h4>3. 明确你需要什么</h4>
  <p>例如：需要确认是否是已知问题、需要恢复某个功能、需要修正文档、需要补充权限说明。不要只抛一个现象让维护者猜。</p>
</div>

<div class="mzk-card">
  <h3>推荐反馈格式</h3>

```text
问题类型：PJSK / 经济系统 / MC / 群管理 / 文档 / 其他
发生时间：2026-xx-xx xx:xx 左右
发生位置：群聊 / 私聊 / 网页 / 具体页面
使用指令：完整粘贴你发送的内容
预期结果：你认为应该发生什么
实际结果：Bot 实际返回、无响应、报错或截图
补充信息：区服、绑定状态、是否刚上传数据、是否最近才出现
```
</div>

<div class="mzk-card mzk-card--blue">
  <h3>更容易被优先处理的反馈</h3>

- 能稳定复现的问题。
- 影响多人或核心功能的问题。
- 有完整日志、截图、指令和时间点的问题。
- 已经排除拼写、权限、区服前缀和数据上传状态的问题。

</div>

<div class="mzk-card mzk-card--yellow">
  <h3>不建议这样做</h3>

- 连续刷屏催修。
- 只说“坏了”“炸了”“不会用”。
- 把多个无关问题塞进同一条反馈。
- 截图遮掉关键内容，导致无法判断 Bot、群号、时间或返回信息。

</div>
